# Langfuse 셀프 호스트 환경에서 오래된 데이터를 정리하는 방법

Langfuse를 셀프 호스트로 운영하면 데이터가 계속 누적된다. 기본 배포만으로는 오래된 데이터를 자동으로 삭제하는 운영 흐름이 바로 준비되지 않기 때문에, 최근 n일만 남기고 나머지를 지우려면 별도의 정리 작업이 필요하다.

예를 들어 최근 30일만 남기고 그 이전 데이터를 삭제하고 싶다면, 단순히 한 테이블만 지워서는 부족하다. Langfuse는 데이터베이스와 오브젝트 스토리지를 함께 사용하므로, 실제로는 여러 저장 위치를 같이 정리해야 한다.

## 무엇을 정리해야 하는가

오래된 Langfuse 데이터를 정리할 때는 보통 두 영역을 함께 봐야 한다.

### 데이터베이스

추적 관련 데이터는 여러 엔터티로 나뉘어 저장된다. 대표적으로 trace, observation, score 같은 데이터가 있다. 오래된 데이터를 삭제할 때 이들을 함께 고려하지 않으면 일부 데이터만 남거나 불필요한 연결 정보가 남을 수 있다.

### 오브젝트 스토리지

셀프 호스트 구성에서는 MinIO나 S3 호환 스토리지를 함께 사용하는 경우가 많다. 이벤트 업로드 파일이나 미디어 파일 같은 데이터가 여기에 저장될 수 있다. 데이터베이스만 지우고 버킷을 지우지 않으면 저장 공간은 계속 증가한다.

즉, 오래된 데이터를 제대로 정리하려면 **DB와 버킷을 함께 정리해야 한다.**

## 왜 별도 정리 작업이 필요한가

기본 셀프 호스트 환경은 Langfuse를 실행하는 데 필요한 웹, 워커, 데이터베이스, 스토리지 구성을 제공한다. 하지만 운영자가 원하는 기준, 예를 들어 최근 30일만 유지하는 정책까지 자동으로 실행해 주는 별도 정리 작업은 직접 구성해야 한다.

그래서 오래된 데이터를 정리하려면 다음과 같은 작업이 필요하다.

- 보존 기간을 정한다.
    
- 오래된 trace, observation, score 등을 찾는다.
    
- 관련된 오브젝트 스토리지 파일도 함께 삭제한다.
    
- 이 작업을 주기적으로 반복 실행한다.
    

## 현실적인 방법

가장 단순한 방법은 **retention 전용 스크립트나 컨테이너를 별도로 두는 것**이다.

이 방식에서는 기존 Langfuse 서비스는 그대로 두고, 정리 전용 프로세스가 별도로 동작한다. 이 프로세스는 Postgres, ClickHouse, MinIO 또는 S3 같은 기존 저장소에 접속해 오래된 데이터를 찾아 지운다.

이 접근의 장점은 분명하다.

- 기존 Langfuse 웹/워커 역할을 건드리지 않는다.
    
- docker compose에 서비스 하나를 추가하는 방식으로 붙일 수 있다.
    
- 보존 일수, 배치 크기, 실행 주기를 따로 관리할 수 있다.
    

## 구성할 때 확인할 점

오래된 데이터를 정리하는 작업은 기존 서비스와 같은 저장소를 정확히 바라봐야 한다. 따라서 다음 정보가 맞아야 한다.

- Postgres 연결 정보
    
- ClickHouse 연결 정보
    
- MinIO 또는 S3 엔드포인트
    
- 이벤트/미디어 버킷 이름과 prefix
    

특히 오브젝트 스토리지 엔드포인트는 컨테이너 내부 기준 주소를 써야 한다. 호스트에서 접근하는 주소와 compose 네트워크 내부 주소가 다를 수 있기 때문이다.

또한 오래된 데이터를 지우는 작업은 삭제 범위를 잘못 잡으면 되돌리기 어렵다. 그래서 처음에는 dry-run 같은 방식으로 실제 삭제 대상이 맞는지 확인하는 편이 안전하다.

## 주기 실행은 어떻게 할 것인가

주기 실행은 외부 스케줄러로 처리할 수도 있고, 정리 스크립트 자체에 반복 실행 기능을 넣을 수도 있다. docker compose 환경에서 retention 전용 컨테이너를 둘 계획이라면, 스크립트 자체가 반복 실행을 지원하면 운영이 단순해진다.

이 경우 컨테이너 하나만 올려두고 보존 기간, 주기, 배치 크기만 환경 변수나 인자로 관리하면 된다.

## 정리

Langfuse 셀프 호스트 환경에서 오래된 데이터를 정리하려면, 한 곳만 보면 안 된다. 데이터베이스와 오브젝트 스토리지를 함께 봐야 한다. 기본 셀프 호스트 배포만으로는 이런 정리 작업이 자동으로 구성되지 않으므로, 별도의 retention 작업을 추가해야 한다.

가장 실용적인 방법은 정리 전용 스크립트나 컨테이너를 따로 두고, 최근 n일만 남기도록 주기적으로 실행하는 것이다.
