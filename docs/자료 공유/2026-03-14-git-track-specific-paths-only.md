# Git에서 특정 경로만 추적하도록 설정하는 방법

Git은 기본적으로 저장소 전체를 관리한다.  
따라서 특정 경로만 추적하려면 `gitignore` 규칙으로 **원하는 경로만 예외 처리**해야 한다.

## 핵심 원리

예를 들어 전체를 무시한 뒤 특정 경로만 허용할 수 있다.

```gitignore
*
!app/
!app/config/
!app/config/**
```

의미는 다음과 같다.

- `*`
    
    - 전체 파일과 디렉터리를 무시한다.
        
- `!app/`
    
    - `app` 디렉터리를 다시 허용한다.
        
- `!app/config/`
    
    - `app/config` 디렉터리를 다시 허용한다.
        
- `!app/config/**`
    
    - `app/config` 아래의 모든 하위 디렉터리와 파일을 다시 허용한다.
        

즉, 이 설정은 `app/config` 경로와 그 하위 내용을 추적 대상으로 남기고 나머지는 무시한다.

## `!app/config/` 와 `!app/config/**` 의 차이

두 패턴은 역할이 다르다.

### `!app/config/`

디렉터리 자체를 무시 대상에서 제외한다.  
즉, Git이 해당 디렉터리 안으로 들어갈 수 있게 한다.

### `!app/config/**`

해당 디렉터리 아래의 모든 하위 디렉터리와 파일을 무시 대상에서 제외한다.  
즉, 실제 내용물까지 추적 가능하게 만든다.

따라서 보통 둘을 함께 사용한다.

```gitignore
*
!app/
!app/config/
!app/config/**
```

## 상위 디렉터리 예외가 필요한 이유

하위 경로만 예외 처리해도 상위 디렉터리가 여전히 무시 상태이면 Git이 그 경로를 탐색하지 못한다.  
그래서 아래처럼 상위 디렉터리부터 순서대로 열어 주어야 한다.

```gitignore
*
!app/
!app/config/
!app/config/**
```

`!app/config/**` 만 단독으로 써서는 의도대로 동작하지 않을 수 있다.

## 이미 추적 중인 파일이 있는 경우

`gitignore` 는 **아직 추적되지 않은 파일**에만 적용된다.  
이미 Git이 추적 중인 파일은 규칙을 바꿔도 자동으로 제외되지 않는다.

이 경우 인덱스를 다시 반영해야 한다.

```bash
git rm -r --cached .
git add .
git commit -m "Reapply gitignore rules"
```

이 명령은 로컬 파일을 지우지 않고 Git 인덱스만 다시 구성한다.

## 정리

Git에서 특정 경로만 추적하려면 다음 원칙을 따르면 된다.

1. 전체를 먼저 무시한다.
    
2. 추적할 상위 디렉터리를 순서대로 예외 처리한다.
    
3. 마지막에 `**` 로 하위 파일 전체를 허용한다.
    
4. 이미 추적 중인 파일이 있다면 인덱스를 다시 반영한다.
