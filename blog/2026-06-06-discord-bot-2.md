# 디스코드 챗봇 v2

약 2년전에 GPT API를 가지고 놀면서 배우기 위해 사람처럼 대화에 참여하는 걸 목적으로 챗봇을 만들었었다.
이름은 [discord-bot-become-human](https://github.com/neuwcodebox/discord-bot-become-human)이다. 당시 디트로이트 비컴 휴먼이라는 게임을 인상 깊게 봤어서 오마주 느낌쓰...
만들고 1년 정도 운영하다가 접었는데 그때와 지금의 지식 수준과 AI 개발 생태계가 많이 달라져서 거기에 맞게 새로 v2를 만들어봤다.

이름은 그대로 쓰고 2만 붙혀서 [discord-bot-become-human-2](https://github.com/neuwcodebox/discord-bot-become-human-2)로 했다. 이름만 비슷하지 그냥 새로 만들었는데 핵심적인 기능들은 다음과 같다.

- Codex 로그인 지원: ChatGPT 구독 계정으로 로그인해서 Codex 할당량을 쓸 수 있다. 즉, 구독료 말고는 돈이 나가지 않는다. 사실 예전에 카카오에서 Pro 구독을 90% 할인해줄 때 샀던 거 써먹으려는 의도가 있었음. 물론 그냥 OpenAI API 연결할 수도 있음.
- OpenClaw 스타일의 하네스: SOUL.md 같은 파일로 봇의 정체성을 설정하거나 USER.md, GROUP.md 파일에 사용자별, 서버별 지식을 장기적으로 기억할 수 있다. 사람이 수동으로 편집해줘도 되지만 Dreaming 기능을 통해서 최근 대화에서 지식을 자동 추출한다. 쉘 도구를 이용해 시스템에 접근할 수 있고 스킬 기능을 이용해 확장할 수 있다. 요즘 AI가 동작하는 환경을 잘 만들어주는 하네스 엔지니어링이라는 말이 또 유행이던데 이런 신조어 남발하는 거 별로 좋아하진 않지만 또 마땅한 용어가 없기도 함...
- 사람처럼 행동: 이게 가장 중요하다. 보통 챗봇을 만든다고 하면 슬래시 명령어로 사람이 직접 부를 때 그 질문에 대해서만 답을 해주는 방식으로 많이들 구현하는데, 이 프로젝트는 그런 방향이 아님. 슬래시 명령어는 전혀 없으며 멘션으로 직접 언급하거나 AI가 판단시 자기를 부르는 것 같거나 자기가 대화에 참여해야 한다고 생각이 들면 능동적으로 참여한다. 대화에서 빠져도 된다 싶을 때에도 스스로 판단해서 대화를 종료한다. 간단한 리액션은 반응 이모지만 달기도 함. 이 모든 동작은 앞에서 말한 SOUL.md 같은 것에 영향을 받는다. 수다스러운 성격이면 반응보단 말로 할거고 뭐 그런식.
- 샌드박스: bwrap을 이용해 명령어를 샌드박스 내에서 실행한다. 어떤 입력이 들어올지 모르는 것과 AI 자체의 불확실성 때문에 이런 보안 조치는 필수적.

핵심 코어는 [pi](https://github.com/earendil-works/pi), [nanobot](https://github.com/HKUDS/nanobot) 프로젝트를 많이 참고했다.
pi는 에이전트 루프에 라이브러리로 사용했고, nanobot은 스킬, 장기기억 부분 등의 소스코드를 참고했다.

그리고 물론 코드는 AI가 짜줬다 ㅋㅋ...
이전에 이미 만든 게 있었으니 요구사항이 명확한지라 작업 시키기가 수월했다.
Codex로 쓰다가 구독 플랜이 끝나서 Claude를 써봤는데 Opus는 토큰을 너무 많이 먹어서 Sonnet으로 했다. 그래서 그런지 결과물의 퀄리티가 Codex(GPT-5.5)보다는 마음에 안 들더라. 앞으로도 Codex를 애용할 듯.
